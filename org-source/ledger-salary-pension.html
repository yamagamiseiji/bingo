Ledger — 給与・年金明細の転記

1 複式簿記による給与明細書の転記

今日は 

サラリーマンなどが 家計簿内で どのように給与明細書を扱うか

の お話し です。しかも 複式簿記 で！

年金受給者も扱う

会社の経理部が社員の給与明細をどのように会社の会計帳簿に記帳するか、ではありません。

2 家計簿での給与明細転記？

複式簿記 で家計簿というアイデアは昔からあります。

わかりやすく説明されているサイトの一つである 「給与収入の仕訳は手取り金額と法定外控除分の二本立て」で紹介されている例を表1に再掲しました。 家計簿での「手取り」金額と「法定外」控除を合わせた仕訳例です。

表1: 手取り金額の仕訳例
日付 借方 金額 貸方 金額 摘要
1/21 妻預金（百五） 335,283 給与収入 335,283 ②
1/21 親睦会PTA会費 9,200 給与収入 9,200 ③
1/21 保険料 15,459 給与収入 15,459 ③
1/21 労働組合費 8,078 給与収入 8,078 ③

上記サイトは、 天引きされる 保険料 や 税金 は自分でコントロールできないので「記入する必要はありません」という立場です。多くの方が賛同され、実践されている一般的な考え方だと思います。

しかしそうだとしても、われわれ一般人がそれを表1のように 借方・貸方 方式の複式簿記にまとめることはとてもハードルが高いです。

3 Ledgerを使ってみると

Ledgerで複式家計簿での給与明細書を転記してみましょう。

3.1 家計簿での給与明細の書き方

Micorosoft 365 サイトの給与管理表 (明細書付き) にあげられている数値例を使わせてもらい、Ledgerで給与明細を複式簿記に沿って転記すると次のようになります。 

2021/10/24 * (株) XYZ
    Income:基本給                         -250,000 JPY
    Income:諸手当                          -20,000 JPY
    ;
    Expenses:社会保険:健康保険                15,000 JPY
    Expenses:社会保険:厚生年金                28,000 JPY
    Expenses:社会保険:雇用保険                 3,000 JPY
    Expenses:社会保険:生命保険                 1,000 JPY
    ;
    Expenses:その他:組合費                      700 JPY
    Expenses:その他:その他                        0 JPY
    ;
    Expenses:租税公課:所得税                   6,500 JPY
    Expenses:租税公課:住民税                  15,000 JPY
    ;
    Assets:積立                             　8,000 JPY
    ;
    Assets:ABC銀行

ソースコード1について補足します：

  • Ledgerでは、貸方・借方の概念を前提とせず、 アカウント （account; 勘定科目）ベースで複式簿記の 原理 を具現化しています。
  • アカウントの第1レベルである Income, Expenses, Assets… などは収入、経費、資産などのように日本語にしても全然 問題ありません。
  • 最終行の Assets:ABC銀行 の行に 金額 が入っていませんが、Ledgerでは一つのトランザクション中の1アカウントだけは ブランク にすることが許容されています。Ledgerが自動的に計算して補完してくれます。
  • Ledgerでは通貨単位（上の例では JPY ）は 必須 です。Ledgerでは１トランザクション内に複数の通貨（JPY, $, &euro; など)を混在させることができます。当日の通貨の交換レートを為替レートデータベースから読み込んで換算してくれます。

reg コマンドの出力は1のようになります。ABC銀行への振込額が補完されています。 

$ ledger reg  -f myledger.dat
2021/10/24 (株) XYZ   Income:基本給             -250,000 JPY   -250,000 JPY
                      Income:諸手当              -20,000 JPY   -270,000 JPY
                      Expenses:社会保険:健康保険    15,000 JPY   -255,000 JPY
                      Expenses:社会保険:厚生年金    28,000 JPY   -227,000 JPY
                      Expenses:社会保険:雇用保険    3,000 JPY   -224,000 JPY
                      Expenses:社会保険:生命保険    1,000 JPY   -223,000 JPY
                      Expenses:その他:組合費         700 JPY   -222,300 JPY
                      Expenses:その他:その他               0   -222,300 JPY
                      Expenses:租税公課:所得税      6,500 JPY   -215,800 JPY
                      Expenses:租税公課:住民税     15,000 JPY   -200,800 JPY
                      Assets:積立                 8,000 JPY   -192,800 JPY
                      Assets:ABC銀行             192,800 JPY            0

この後、自由自在に任意のアカウントを任意の期間で集計したり表示したりできます。 たとえば、2019年1月以降の住民税の合計を見たいときには 

$ ledger bal 地方税 -b 2019/01
       1,203,300 JPY  Expenses:租税公課:地方税

あと、これをきれいなグラフに書いて可視化するのも　おちゃのこサイサイです。

4 年金の支給明細のLedger転記例

つぎに複式簿記の家計簿で年金支給を記帳する例をあげます。金額はあてずっぽうです。 

2021/10/15 * 国民年金
   Income:国民年金             (-100,000 JPY /2) ; [2021/08/15]
   Income:国民年金             (-100,000 JPY /2) ; [2021/09/15]
   ;
   Expenses:社会保険:介護保険     (15,000 JPY /2) ; [2021/08/15]
   Expenses:社会保険:介護保険     (15,000 JPY /2) ; [2021/09/15]
   ;
   Expenses:租税公課:地方税       (20,000 JPY /2) ; [2021/08/15]
   Expenses:租税公課:地方税       (20,000 JPY /2) ; [2021/09/15]
   ;
   Assets:ABC銀行

ソースコード2について補足します：

  • 年金は2ヶ月に1回、偶数月の15日にそれよりも前の 2ヶ月分 が支給されます。各行の右端の [ ] カッコ内の日付がそれを表現しています。支給日は10月15日ですが、実際には8月分と9月分の収入となることをLedger風に表現しています。
  • ( ) で囲まれた金額欄内には2で割る演算が書かれています。
  • このLedgerファイルの reg コマンド出力を見ると、次のようにちゃんと年金額と控除金額が8月分と9月分に振り分けられています。
$ ledger reg -f myledger.dat
2021/08/15 国民年金  Income:国民年金            -50,000 JPY    -50,000 JPY
                                    Expenses:社会保険:介護保険    7,500 JPY    -42,500 JPY
                                    Expenses:租税公課:地方税      10,000 JPY   -32,500 JPY
2021/09/15 国民年金   Income:国民年金            -50,000 JPY    -82,500 JPY
                                     Expenses:社会保険:介護保険    7,500 JPY    -75,000 JPY
                                     Expenses:租税公課:地方税     10,000 JPY    -65,000 JPY
2021/10/15 国民年金   Assets:ABC銀行              65,000 JPY
  • 最終行の Assets:ABC銀行 の日付は、銀行振込みの日付＝10月15日になっています。

5 こうすることのメリット

  • 控除 または 特別徴収 というと聞こえは良いですが、要するに有無を言わさず 天引き される 税金 や 保険料 をきちんと分析対象とすることができます。

6 おまけ

6.1 定期的なトランザクションを用意する

~/ledger-schedule.ledger ファイルに次のようなエントリを書いておきましょう。 [*/E/15] は毎偶数月（even）の15日を意味します。 

[*/E/15] 国民年金
    Income:国民年金      (-100,000 JPY /2) ; [2021/xx/15]
    Income:国民年金      (-100,866 JPY /2) ; [2021/yy/15]
    ;
    Expenses:社会保険:介護保険 (15,000 JPY /2) ; [2021/xx/15]
    Expenses:社会保険:介護保険 (12,700 JPY /2) ; [2021/yy/15]
    ; 
    Expenses:租税公課:地方税   (20,000 JPY /2) ; [2021/xx/15]
    Expenses:租税公課:地方税   (20,000 JPY /2) ; [2021/yy/15]
    ;
    Assets:ABC銀行

そうすると、Emacs でLedgerファイルを開いている際に、 C-c, u を叩くと、年金支払い日の前になると、 [*/E/15] のところに具体的な日付が入ったトランザクション例が表示されます。それをそのままコピペして修正が必要な箇所だけを直せばよい、という風に省力化されます。
